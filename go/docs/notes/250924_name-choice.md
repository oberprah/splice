# Why "Splice"?

## The Name Choice for Our Git TUI Tool

After exploring various naming options for our git terminal user interface application, we settled on **"splice"** for several compelling reasons:

## Core Reasoning

### 1. Technical Relevance
- **Code Integration**: In programming, splicing refers to joining or modifying code segments, which aligns perfectly with git's core function of managing code changes
- **Array Operations**: The `splice()` method in programming languages like JavaScript modifies arrays by adding, removing, or replacing elements - similar to how git operations modify the codebase history

### 2. Git Workflow Alignment
- **Merging Concepts**: Splicing traditionally means joining two things together, which mirrors git's merge operations
- **Change Management**: The tool focuses on viewing diffs and logs - essentially examining how code has been "spliced" together over time
- **Branch Navigation**: Moving between branches is like navigating different splice points in the codebase evolution

### 3. Practical Benefits
- **Memorable**: Short, punchy name that's easy to remember and type
- **Available**: Less likely to conflict with existing tools since it doesn't use obvious git terminology
- **Clean**: Avoids the crowded "git*" namespace while maintaining conceptual connection

### 4. Future-Proof
- **Extensible**: The name works well for current features (logs, diffs) and future enhancements
- **Professional**: Sounds technical and purposeful without being overly specific
- **Brandable**: Distinctive enough to build recognition around

## The Decision Process

We initially considered names like `gitui`, `gittree`, and `gitview`, but decided to move away from explicit "git" terminology. From alternatives like `trunk`, `delta`, `chronicle`, and others, "splice" emerged as the winner because it:

1. Captures the essence of what the tool does (examining code joins/changes)
2. Sounds professional and technical
3. Is short and memorable
4. Has clear metaphorical connections to git operations

## Conclusion

"Splice" represents the intersection of technical accuracy and practical usability - a name that developers will understand intuitively while being distinctive enough to stand out in the ecosystem of git tools.
